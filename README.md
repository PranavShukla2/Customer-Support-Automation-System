# Customer Support Automation System

A multi-agent customer support automation system built using [LangChain](https://python.langchain.com/) and [LangGraph](https://python.langchain.com/docs/langgraph/). This system intelligently routes user queries to specialized department agents, utilizes Retrieval-Augmented Generation (RAG) for accurate responses, maintains conversation history, and includes a human-in-the-loop mechanism for sensitive operations.

## Architecture Overview

The system models the customer support flow as a state machine (`StateGraph`) using LangGraph. The core components are:

1. **State Management (`state.py`)**: 
   Maintains the `SupportState`, tracking the conversation history, detected intent, retrieved context, draft responses, and approval status.
   
2. **Intent Classification & Routing (`router.py`)**: 
   Analyzes the customer's incoming message and classifies it into one of five intents: `Sales`, `Technical`, `Billing`, `Account`, or `Memory`. It routes the execution to the corresponding department node.

3. **Department Agents (`agents.py` & `memory.py`)**:
   - **Sales, Technical, Billing, Account**: Each agent represents a specialized department. They receive the query, pull in relevant context (RAG simulation), and generate a draft response. The `Billing` agent monitors for sensitive keywords (e.g., "refund", "cancel") and flags the state as requiring human approval (`requires_approval = True`).
   - **Memory**: Specifically handles queries regarding past conversation history.

4. **Supervisor & Human Review (`supervisor.py`)**:
   - **Human Review**: A conditional node that pauses the graph execution (interrupt) when a drafted response requires approval. A human can review and approve before the workflow continues.
   - **Supervisor**: The final agent in the chain that refines and polishes the draft response generated by the department agents to ensure a warm, helpful, and professional customer-facing reply.

5. **Graph Orchestration & Persistence (`main.py`)**:
   Constructs the nodes and edges of the `StateGraph` and defines the interrupts. It also integrates a SQLite checkpointer (`support_memory.db`) to persist conversation memory across threads.

## Setup Guide

### 1. Prerequisites
Ensure you have Python 3.9+ installed. Install the required dependencies:

```bash
pip install -r requirements.txt
```

*(Requirements include `langgraph`, `langchain-core`, `langchain-ollama`, `pydantic`, and `langgraph-checkpoint-sqlite`)*

### 2. Local Setup via Ollama (Default)
By default, the project is configured to run entirely locally using [Ollama](https://ollama.com/) with the `llama3.1` model.

1. **Install Ollama**: Download and install from [ollama.com](https://ollama.com/).
2. **Pull the Model**: Open your terminal and run the following command to download the model:
   ```bash
   ollama pull llama3.1
   ```
3. **Run the Application**: Ensure the Ollama server is running, then start the demo:
   ```bash
   python main.py
   ```

### 3. Alternative Setup via OpenAI
If you prefer to use OpenAI's models (e.g., GPT-4o), you can easily modify the codebase to swap out the LLM provider.

1. **Install the OpenAI package**:
   ```bash
   pip install langchain-openai
   ```
2. **Set your API Key**:
   ```bash
   export OPENAI_API_KEY="your-api-key-here"
   ```
3. **Update the Code**:
   In `router.py`, `agents.py`, `memory.py`, and `supervisor.py`, replace the `ChatOllama` import and instantiation:
   ```python
   # Remove this:
   # from langchain_ollama import ChatOllama
   # llm = ChatOllama(model="llama3.1", temperature=0)

   # Add this:
   from langchain_openai import ChatOpenAI
   llm = ChatOpenAI(model="gpt-4o", temperature=0)
   ```

## Demonstration Queries & Routing Paths

The `main.py` file includes a built-in demo that executes a set of representative queries. The table below illustrates how the LangGraph handles these queries, including standard flows and the human-in-the-loop interruption.

| # | Demonstration Query | Expected Intent | Routing Path |
|---|--------------------|-----------------|--------------|
| 1 | *"What pricing plans do you offer?"* | **Sales** | `Classifier` → `Sales Agent` → `Supervisor` → `END` |
| 2 | *"My API calls are timing out on the batch endpoint."* | **Technical** | `Classifier` → `Technical Agent` → `Supervisor` → `END` |
| 3 | *"How do I reset my password?"* | **Account** | `Classifier` → `Account Agent` → `Supervisor` → `END` |
| 4 | *"I'd like a refund for last month's charge."* | **Billing** | `Classifier` → `Billing Agent` → ⏸️ **`Human Review (Interrupt)`** → `Supervisor` → `END` |
| 5 | *"What was my previous issue?"* | **Memory** | `Classifier` → `Memory Agent` → `Supervisor` → `END` |

*(Note: Query 4 triggers an interrupt because it contains the keyword "refund". Execution pauses until `is_approved` is set to True by a human operator.)*
